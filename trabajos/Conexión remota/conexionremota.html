RESUMEN

Se ha implementado y probado acceso remoto y administración entre tres máquinas virtuales conectadas dentro de una red local. Se ha trabajado con una VM Linux (IP 192.168.1.1), una segunda VM Linux (IP 192.168.1.2) y una VM Windows (IP 192.168.1.3).
Las tareas incluyeron la configuración de servicios SSH y pruebas de escritorio remoto (RDP), así como la instalación y uso de software de acceso remoto como AnyDesk.
Las conexiones se documentaron utilizando usuarios personalizados y capturas en cada paso relevante del proceso.

Conexión remota por SSH en Linux

  1. Crear usuario personalizado $ sudo adduser nombreapellido
  2. Instalar OpenSSH Server
    $ sudo apt update
    $ sudo apt install openssh-server # Intalamos openssh
    $ sudo systemctl start ssh #Iniciamos servicio ssh
    $ sudo systemctl enable ssh #Habilitado el servicio ssh
  3. Imagen (Status servicio ssh)
    4. Estado del servicio SSH en el sistema Linux.

  4. Probamos conexión SSH $ ssh nombreapellido@ip_del_servidor
    # ip del servidor 192.168.1.1
    5. Ejemplo de conexión SSH desde Linux.

  5. Creación de una carpeta por SSH
    6. Creación y visualización de carpetas por SSH.

Acceso SSH sin contraseña

  1. Crear clave pública y privada en el cliente Linux. ¡La clave privada no se comparte!
    $ ssh-keygen
    2. Generación de claves SSH públicas y privadas en Linux.

  2. Copiar la clave pública al servidor remoto $ ssh-copy-id julioluengo@192.168.1.1
    3. Transferencia de la clave pública mediante SSH.

  3. Probar la conexión sin la contraseña
    4. Acceso a través de SSH usando clave pública.
